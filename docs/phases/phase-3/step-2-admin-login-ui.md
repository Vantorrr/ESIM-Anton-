## Что нужно сделать

- найти существующий backend `/api/auth/login` contract;
- добавить login form/session state в admin;
- сохранить token безопасно в рамках текущей frontend-архитектуры;
- отделить PIN как optional локальный барьер, если он нужен бизнесу.
